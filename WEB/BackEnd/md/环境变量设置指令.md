# 🎯 环境变量设置指令

## 核心概念
设置环境变量后，应用会自动：
- 识别开发/生产环境
- 使用正确的数据库、邮件配置
- 无需在代码中硬编码敏感信息

---

## 🚀 配置方法（macOS/Linux专属）

### macOS 开发环境设置（永久有效）

```bash
nano ~/.zshrc

export DATABASE_PASSWORD=你的数据库密码
export MAIL_PASSWORD=你的邮箱授权码
export SPRING_PROFILES_ACTIVE=dev

source ~/.zshrc
```

### Linux 生产环境设置（永久有效）

```bash
sudo nano /etc/environment

DATABASE_URL=jdbc:mysql://你的数据库服务器:3306/ZyroFrp
DATABASE_USERNAME=zyrofrp_user
DATABASE_PASSWORD=生产环境数据库密码
MAIL_HOST=smtp.163.com
MAIL_USERNAME=生产环境邮箱@163.com
MAIL_PASSWORD=生产环境邮箱授权码
MAIL_PORT=465
SERVER_PORT=8085
SPRING_PROFILES_ACTIVE=prod
```

## 🔧 验证环境变量

### 检查应用启动日志
启动应用后，查看日志中是否显示：
```
The following 1 profile is active: "dev"
```

## 🎯 关键环境变量说明

- `SPRING_PROFILES_ACTIVE` = `dev`(开发) / `prod`(生产)
- `DATABASE_PASSWORD` = 你的数据库密码
- `MAIL_PASSWORD` = 你的邮箱授权码

## 📝 注意事项

- **macOS**：环境变量设置在 `~/.zshrc` 文件中，永久有效
- **Linux**：环境变量设置在 `/etc/environment` 文件中，系统级永久有效
- 配置完成后需要重新加载配置或重新登录系统
- 生产环境只需将 `SPRING_PROFILES_ACTIVE` 改为 `prod`
